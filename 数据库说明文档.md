# 数据库说明文档

## 1. 概述

本项目使用 **SQLite** 存储用户拼图行为与报告生成日志，用于：

1. 记录用户会话与动作轨迹；
2. 统计用户近期行为特征（用时、步数、犹豫比例、回退比例、趋势等）；
3. 将近期行为摘要拼接到提示词中，提升心理分析报告的稳定性与一致性。

数据库实现位于 [analytics_store.py](F:/心宇宙1/backend/analytics_store.py)。

---

## 2. 数据库位置与初始化

- 数据库文件：`backend/data/behavior_analytics.db`
- 初始化入口：后端启动时在 [app.py](F:/心宇宙1/backend/app.py) 创建 `AnalyticsStore`

## 3. 表结构

### 3.1 users（用户基础表）

用于记录用户首次与最近活跃时间。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `client_id` | TEXT | PK | 用户标识（前端生成并归一化） |
| `first_seen` | REAL | NOT NULL | 首次出现时间戳（Unix 秒） |
| `last_seen` | REAL | NOT NULL | 最近活跃时间戳（Unix 秒） |

---

### 3.2 game_sessions（拼图会话表）

一局拼图的聚合状态快照（可反复更新）。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | INTEGER | PK AUTOINCREMENT | 自增主键 |
| `game_id` | TEXT | UNIQUE NOT NULL | 拼图局 ID（服务端生成） |
| `client_id` | TEXT | NOT NULL | 用户标识 |
| `image_source` | TEXT |  | 图片来源 |
| `grid_size` | INTEGER |  | 难度（n x n 的 n） |
| `modifiers_json` | TEXT |  | 词条配置 JSON（旋转/隐藏/捣蛋鬼） |
| `started_at` | REAL |  | 开始时间戳 |
| `ended_at` | REAL |  | 结束时间戳（完成时写入） |
| `game_state` | TEXT |  | `playing / completed / ...` |
| `move_count` | INTEGER | DEFAULT 0 | 步数 |
| `progress` | REAL | DEFAULT 0 | 完成进度百分比 |
| `completion_time_seconds` | INTEGER |  | 完成用时（秒） |
| `piece_order_json` | TEXT |  | 首次放置顺序 JSON |
| `time_intervals_json` | TEXT |  | 动作间隔数组 JSON |
| `modification_count` | INTEGER | DEFAULT 0 | 修改次数 |
| `created_at` | REAL | NOT NULL | 记录创建时间戳 |
| `updated_at` | REAL | NOT NULL | 记录更新时间戳 |

索引：
- `idx_game_sessions_client_updated(client_id, updated_at DESC)`

---

### 3.3 action_logs（动作明细表）

逐条记录用户动作日志。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | INTEGER | PK AUTOINCREMENT | 自增主键 |
| `game_id` | TEXT | NOT NULL | 拼图局 ID |
| `client_id` | TEXT | NOT NULL | 用户标识 |
| `action` | TEXT | NOT NULL | 动作名（如 `place_from_tray`） |
| `payload_json` | TEXT |  | 动作参数 JSON |
| `move_count` | INTEGER |  | 动作后步数 |
| `progress` | REAL |  | 动作后进度 |
| `game_state` | TEXT |  | 动作后局面状态 |
| `elapsed_seconds` | INTEGER |  | 动作后累计耗时（秒） |
| `created_at` | REAL | NOT NULL | 动作时间戳 |

索引：
- `idx_action_logs_client_time(client_id, created_at DESC)`

---

### 3.4 report_logs（报告日志表）

记录报告生成时的输入提示词与输出文本。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | INTEGER | PK AUTOINCREMENT | 自增主键 |
| `client_id` | TEXT | NOT NULL | 用户标识 |
| `game_id` | TEXT |  | 拼图局 ID |
| `image_source` | TEXT |  | 图片来源 |
| `prompt_text` | TEXT |  | 本次发送给模型的用户提示词文本 |
| `report_text` | TEXT |  | 模型返回报告文本 |
| `created_at` | REAL | NOT NULL | 报告生成时间戳 |

索引：
- `idx_report_logs_client_time(client_id, created_at DESC)`

## 4. 数据流转（写入时机）

### 4.1 创建拼图局

- API：`POST /api/puzzle/games`
- 逻辑：
  1. 后端创建拼图局状态；
  2. 调用 `upsert_game_session(...)` 写入/更新 `game_sessions`；
  3. 同步更新 `users.last_seen`。

### 4.2 执行动作

- API：`POST /api/puzzle/games/<game_id>/actions`
- 逻辑：
  1. 后端应用动作并更新状态；
  2. `log_action(...)` 写入 `action_logs`；
  3. `upsert_game_session(...)` 更新当前会话快照；
  4. 同步更新 `users.last_seen`。

### 4.3 生成心理报告

- API：`POST /api/generate-report`
- 逻辑：
  1. `update_from_report_payload(...)` 把前端上报数据同步到 `game_sessions`；
  2. `build_recent_behavior_prompt(...)` 读取最近 14 天行为并生成“近期行为摘要”；
  3. 摘要拼入提示词发送给模型；
  4. `save_report(...)` 记录 `prompt_text` 与 `report_text` 到 `report_logs`。

---

## 5. 近期行为摘要生成规则

统计窗口默认：
- 最近 `14` 天
- 最多 `12` 局会话

主要统计项（来自 `game_sessions` + `action_logs`）：
- 完成局数 / 总局数
- 平均用时、中位用时
- 平均步数、平均修改次数
- 平均操作间隔
- 高犹豫动作占比（间隔 >= 4 秒）
- 首次放置覆盖量均值
- 常用难度、常用图片
- 回退动作占比、翻转动作占比
- 趋势判断（新旧半段对比用时与步数变化）

当样本不足时，摘要会显式给出“样本不足/数据不足”。

---

## 6. client_id 说明

- 前端在浏览器 `localStorage` 生成并持久化 `clientId`；
- 后端通过 `normalize_client_id(...)` 做归一化：
  - 只保留 `a-zA-Z0-9._:-`
  - 为空时使用 `anonymous`
  - 最长 128 字符

## 7. 维护与运维建议

### 7.1 备份

建议定时复制：
- `backend/data/behavior_analytics.db`

### 7.2 清理（建议）

当前代码未自动清理历史数据，建议后续增加：
- 定期删除超过 N 天的 `action_logs` 与 `report_logs`
- 会话归档策略（保留完成局，清理长期未完成局）

### 7.3 隐私建议

当前 `client_id` 为匿名设备标识，建议进一步：
- 对 `client_id` 做单向哈希后再落库；
- 增加“按 client_id 删除历史数据”的接口；
- 报告日志可配置采样或脱敏落库策略。

## 8. 常用查询示例

### 8.1 查看最近会话

```sql
SELECT game_id, client_id, game_state, move_count, progress, updated_at
FROM game_sessions
ORDER BY updated_at DESC
LIMIT 20;
```

### 8.2 查看某用户最近动作

```sql
SELECT action, payload_json, move_count, progress, created_at
FROM action_logs
WHERE client_id = 'your-client-id'
ORDER BY created_at DESC
LIMIT 50;
```

### 8.3 查看某用户报告生成记录

```sql
SELECT id, game_id, image_source, created_at
FROM report_logs
WHERE client_id = 'your-client-id'
ORDER BY created_at DESC;
```
