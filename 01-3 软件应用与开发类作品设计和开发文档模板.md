# 第一章 需求分析

1.1.1 开发背景

 

即时战略（Real-Time Strategy，简称RTS）游戏是电子游戏领域中历史悠久且极具深度的游戏类型。从早期的《命令与征服》《星际争霸》到近年的《英雄连》《战争游戏》系列，RTS游戏以其独特的战术策略性和实时操作体验吸引了大量玩家。然而，传统RTS游戏往往采用简化的伤害计算模型，单位之间的对抗缺乏真实感和战术深度。

 

与此同时，以《战争雷霆》《坦克世界》为代表的载具模拟游戏则采用了高度拟真的装甲穿透计算系统，考虑装甲厚度、入射角度、弹种特性等多重因素，为玩家带来了真实的装甲对抗体验。但这类游戏通常是第一人称或第三人称视角的动作游戏，缺乏RTS游戏的宏观战略层面。

 

1.1.2 开发动机

 

本项目的核心创意在于将载具模拟游戏中成熟的穿甲计算系统与RTS游戏的宏观战略玩法相结合，创造一种全新的游戏体验。玩家不仅需要考虑传统RTS游戏中的兵种克制、地形利用等策略要素，还需要关注每一辆载具的装甲布局、弱点位置、进攻角度等战术细节。

 

开发本作品的具体动机包括：

 

填补市场空白：目前市场上鲜有将精细穿甲计算与RTS玩法深度结合的作品，本项目旨在探索这一创新方向。

 

技术能力验证：通过完整开发一款具有一定复杂度的游戏作品，验证和提升Unity引擎应用、游戏系统设计、算法实现等多方面的技术能力。

 

学术实践结合：将软件工程课程中学习的需求分析、系统设计、测试方法等理论知识应用于实际项目开发中。

 

1.1.3 项目定位

 

本项目定位为一款单机即时战略游戏，主打以下特色：

 

模块化伤害系统：每辆载具由多个独立模块组成（炮塔、车体、履带、弹药架、乘员舱等），不同模块被击中会产生不同效果，如弹药架殉爆、乘员阵亡、履带断裂等。

 

拟真穿甲计算：引入入射角度、等效装甲、跳弹判定、穿深衰减等真实物理要素，鼓励玩家精准射击敌方弱点而非简单的火力覆盖。

 

战术深度：地形遮蔽、侧翼包抄、集火打击等战术在本游戏中具有实际意义，为玩家提供丰富的策略选择空间。

1.2.2 差异化优势

 

本项目相比竞品的核心优势在于将《战争雷霆》级别的穿甲计算简化后引入RTS游戏框架，同时保持《英雄连》式的多单位指挥体验，创造了一种独特的游戏类型。

 

1.3 目标用户群体

 

策略游戏爱好者：喜欢《星际争霸》《英雄连》等RTS游戏的玩家，追求战术策略和操作体验的结合。

 

军事题材爱好者：对现代装甲载具感兴趣的玩家，希望在游戏中体验真实的装甲对抗。

 

硬核游戏玩家：追求游戏深度和可玩性，愿意投入时间学习游戏机制的玩家。

 

1.4 功能需求

 

1.4.1 核心功能模块

 

单位管理模块：负责游戏中所有装甲载具的管理，包括载具实例的创建、初始化和销毁管理；载具属性系统（生命值、装甲值、速度等）；载具状态管理（待机、移动中、交战中、已摧毁）；模块化组件系统，每辆载具由炮塔、车体、履带、弹药架、乘员等独立模块组成。

 

战斗系统模块：实现完整的装甲对抗机制，包括武器开火系统、穿甲计算系统、伤害应用系统、占领区域系统。

 

用户交互模块：提供玩家与游戏的所有交互接口，包括选择系统（点选、框选、追加选择）、命令系统（移动、攻击）、相机控制系统（平移、缩放）。

 

AI系统模块：为游戏提供单机对战的敌方智能，包括自动寻敌、状态机行为决策、路径规划、威胁评估。

 

UI系统模块：提供游戏的所有界面展示，包括战斗HUD、部署界面、准备界面、结算界面。

 

特效系统模块：管理游戏中的所有视觉效果，包括开火特效、命中特效、爆炸特效。

1.5 非功能需求

 

性能需求：在推荐配置下，战斗场景（20个活动单位）应保持流畅运行体验；场景加载时间应控制在合理范围内。

 

可维护性需求：系统采用模块化架构，各功能模块之间低耦合、高内聚；代码按功能域组织在不同命名空间下（Core、Units、Battle、UI等）。

 

可扩展性需求：支持方便地添加新的载具类型和武器系统，只需配置JSON数据文件和预制体。

 

用户体验需求：遵循RTS游戏的通用操作习惯，降低学习成本；所有玩家操作都有明确的视觉反馈。

 

1.6 技术选型

 

开发引擎：Unity 2022.3 LTS，成熟的3D游戏引擎，组件化开发模式适合游戏开发。

 

编程语言：C，Unity的主要脚本语言，面向对象，与引擎深度集成。

 

数据存储：JSON文件，存放于StreamingAssets目录，可读，无需额外数据库依赖。

 

版本控制：Git + GitHub。

 

开发工具：Visual Studio 2022 + Unity Editor。

# 第三章 详细设计

3.1 界面设计

 

3.1.1 主战斗界面

 

主战斗界面是玩家进行游戏操作的核心界面，采用经典RTS游戏的布局设计。

 

视角控制：屏幕中央为3D战场的俯视视图，玩家可通过以下方式控制视角：

上下左右键：平移相机位置，查看战场不同区域

鼠标滚轮：缩放视角，拉近或拉远观察距离

屏幕边缘：鼠标移动到屏幕边缘时自动平移视角

 

单位操作：

左键单击：选中单个己方载具，显示其详细信息

左键拖拽：绘制选择框，框选区域内所有己方载具

Shift+左键：追加选择，将新单位添加到当前选择

右键单击地面：对选中单位下达移动命令

A键+左键单击敌方：对选中单位下达攻击命令

 

信息显示：

顶部状态栏：显示当前游戏时间、游戏状态（部署中/战斗中）、占领区进度

底部信息面板：选中载具后显示详细信息，包括载具名称、类型、血量、各模块状态、乘员存活数

右下角小地图：实时显示战场俯视图，友方单位蓝色标记，敌方单位红色标记

 

3.1.2 部署界面

 

部署界面在战斗开始前呈现，允许玩家配置己方兵力。

 

单位列表区：界面左侧显示可部署的单位列表，每个单位条目显示载具图标、名称、类型和基础属性预览。

 

部署区域：中央战场视图中，出生区域以绿色高亮显示，玩家只能在此区域内放置单位。点击单位列表中的载具，再点击部署区域内的位置即可放置。

 

开始按钮：部署完成后，点击"开始战斗"按钮进入战斗阶段。

 

3.1.3 结算界面

 

战斗结束后弹出结算界面，显示本场战斗的详细统计：战斗结果（胜利/失败）、击杀敌人数量、己方损失数量、占领区域数量、战斗持续时间，以及根据战斗表现获得的资源奖励。

3.2 数据存储设计

 

3.2.1 数据存储方案

 

本系统采用JSON文件存储游戏配置数据，所有数据文件存放于Unity项目的StreamingAssets目录下。JSON格式人类可读，便于调试和修改，Unity提供内置的JSON序列化支持，无需额外的数据库依赖。

 

3.2.2 数据文件结构

 

载具数据 (VehicleData.json)：存储所有载具类型的基础配置，包含vehicleId（载具唯一标识）、vehicleName（载具名称）、vehicleClass（载具类型）、nation（所属国家）、crewCount（乘员数量）、maxHealth（最大血量）、armorData（装甲配置）、mobilityData（机动性配置）、weaponData（武器配置）。

 

炮弹数据 (ShellData.json)：存储所有弹种的属性配置，包含shellId、shellName、shellType（弹种类型：AP/HEAT/HE）、basePenetration（基础穿深）、muzzleVelocity（炮口初速）、penetrationDropoff（穿深衰减系数）、postPenDamage（后效伤害值）。

 

模块数据 (ModuleData.json)：存储载具各模块的配置，包含moduleId、moduleType（Turret/Hull/Track/Ammo/Crew）、baseHealth、armorThickness、criticalMultiplier、disableEffect。

 

特效配置 (EffectsConfig.json)：存储各类特效的配置，包含effectType、prefabPath、duration、scale。

4.5 问题修正记录

 

问题1：框选时偶尔选中屏幕外单位。修正：增加视锥体裁剪判定。

 

问题2：穿甲计算在极端角度时结果异常。修正：添加角度范围限制和边界值处理。

 

问题3：多个载具同时开火时性能下降。修正：采用对象池技术优化特效实例化。

 

问题4：打包后MeshCollider失效导致单位无法被选中。修正：将MeshCollider改为BoxCollider。

 

问题5：打包后射线检测命中CaptureZone触发器而非单位。修正：射线检测添加QueryTriggerInteraction.Ignore参数。

 

问题6：打包后结算界面不显示。修正：在EndGame方法中直接调用SettlementUI.ShowSettlement作为保障。

 

问题7：BattleManager只注册部分占领区域。修正：改用FindObjectsOfType自动查找所有CaptureZone。

 

问题8：敌方单位被消灭后AliveEnemy计数不更新。修正：确认Status正确设置为Destroyed。

 

问题9：炮塔销毁特效报空引用异常。修正：添加Rigidbody组件存在性检查。

 

问题10：部署界面单位名称不显示。修正：将Text组件改为TextMeshProUGUI。

# 第五章 安装及使用

5.1 运行环境要求

 

最低配置：Windows 10 64位，Intel Core i5，8GB RAM，GTX 1050，2GB存储空间

 

推荐配置：Windows 11 64位，Intel Core i7，16GB RAM，GTX 1660

 

5.2 安装过程

 

本游戏采用Unity引擎直接打包生成可执行文件，安装过程极其简单：

 

\1. 获取游戏压缩包（ForbiddenZone_v1.0.zip）

 

\2. 将压缩包解压到任意目录

 

\3. 双击运行ForbiddenZone.exe启动游戏

 

无需额外安装运行库或依赖项，解压即可运行。

 

5.3 典型使用流程

 

游戏启动：双击ForbiddenZone.exe启动游戏，进入主菜单后点击"开始游戏"。

 

部署阶段：在左侧单位列表中点击想要部署的载具，在中央战场的绿色部署区域内点击放置，完成后点击"开始战斗"。

 

战斗阶段：

 

视角控制：上下左右键平移，鼠标滚轮缩放

 

单位控制：左键点击/拖拽选择，右键移动，A键+左键攻击

 

胜利条件：消灭所有敌方单位或占领所有战略区域
